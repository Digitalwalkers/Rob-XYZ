# 机器人时序数据上传与分析平台 — 需求总结

## 背景

机器人设备持续产生传感器与状态数据，现场工程师需要将采集到的 CSV 日志文件上传至平台，快速回溯设备运行状况、排查异常。

---

## 数据格式

每行代表某台机器人在某一时刻的状态快照：

| 字段 | 类型 / 说明 |
|---|---|
| `robot_id` | 机器人唯一标识，如 `robot_001` |
| `timestamp` | 采集时间，ISO 8601 格式，如 `2024-03-15T14:23:01Z` |
| `location_x` / `location_y` | 机器人当前坐标 |
| `battery_level` | 电池电量百分比（0–100） |
| `device_a_status` / `device_b_status` | 设备状态，枚举值：`ok` / `warning` / `error` |
| `speed` | 当前移动速度（m/s） |
| `error_code` | 错误码，无异常时为空 |

> 字段命名允许轻微差异，但数据类型和语义应一致。

---

## 功能需求

### 1. 文件管理
- 上传 CSV 日志文件
- 查看历史上传记录
- 删除不再需要的文件
- 上传时对文件做基本校验，避免无效数据进入系统

### 2. 数据查看与分析（核心模块）

用户选中某个已上传文件后，需要能够：

- **浏览原始数据**：以某种方式查看文件中的记录内容
- **筛选与聚焦**：按条件缩小关注范围，例如：
  - 某台机器人
  - 某个时间段
  - 某种异常状态
- **获得有用的洞察**：基于时序与设备状态的特点，提供对工程师实际有价值的分析视角

> 具体如何组织功能、呈现哪些指标、采用什么交互形式，由开发者决定。
> 评审更关注对「工程师用这个平台想解决什么问题」的理解，而不是功能点数量。
